## 2. 環境描述

### 2.1 格子世界（Cliff Walking）

採用 **4 × 12** 矩形網格，遵循 Sutton & Barto 教科書設定：

```
Row 0 →  [  ][ ][ ][ ][ ][ ][ ][ ][ ][ ][ ][ ]
Row 1 →  [  ][ ][ ][ ][ ][ ][ ][ ][ ][ ][ ][ ]
Row 2 →  [  ][ ][ ][ ][ ][ ][ ][ ][ ][ ][ ][ ]
Row 3 →  [ S][C][C][C][C][C][C][C][C][C][C][ G]
              ←————————  Cliff  ————————→
```

| 元素 | 座標 | 說明 |
|------|------|------|
| Start（S） | `(3, 0)` 左下角 | 每回合起始位置 |
| Goal（G） | `(3, 11)` 右下角 | 到達即結束回合 |
| Cliff（懸崖） | `(3, 1)` ~ `(3, 10)` | 掉入給 −100 並重置 |

### 2.2 動作空間

| 動作編碼 | 方向 |
|---------|------|
| 0 | ↑ UP |
| 1 | → RIGHT |
| 2 | ↓ DOWN |
| 3 | ← LEFT |

邊界碰撞：嘗試超出格子邊界時，代理留在原地。

### 2.3 獎勵機制

| 事件 | 獎勵 |
|------|------|
| 正常移動一步 | −1 |
| 掉入懸崖 | −100，代理回到 Start |
| 到達 Goal | 回合結束（計 −1） |

## 3. 演算法規格

### 3.1 共通設定

- **策略**：ε-greedy
- **Q 表初始化**：全為 0
- **超參數**（作業標準參數）：

| 參數 | 符號 | 值 |
|------|------|-----|
| 學習率 | α | 0.5 |
| 折扣因子 | γ | 0.9 |
| 探索率 | ε | 0.1 |
| 訓練回合數 | Episodes | 500 |
| 獨立重複次數 | Runs | 50 |

> 注意：作業說明建議 α=0.1，但為了與 Sutton & Barto 參考圖一致，本實作採用 α=0.5。

### 3.2 Q-learning 規格（Off-policy）

**更新公式**：

```
Q(s, a) ← Q(s, a) + α · [r + γ · max_a' Q(s', a') − Q(s, a)]
```

**特點**：
- 使用下一狀態所有動作中**最大的 Q 值**進行更新
- 更新與實際執行的行為策略**無關**（Off-policy）
- 學到的策略是理論最優（貼崖捷徑），但訓練中因探索常掉崖

**程式碼位置**：`rl_algorithms.py` → `q_learning()`

### 3.3 SARSA 規格（On-policy）

**更新公式**：

```
Q(s, a) ← Q(s, a) + α · [r + γ · Q(s', a') − Q(s, a)]
```

其中 `a'` 是由 ε-greedy **實際選出的下一動作**

**特點**：
- 使用**實際執行的下一動作**對應的 Q 值進行更新
- 更新反映探索策略的影響（On-policy）
- 學到的策略較保守（上排繞路），訓練更穩定

**程式碼位置**：`rl_algorithms.py` → `sarsa()`

## 9. 預期實驗結論

根據 Sutton & Barto 理論與實驗結果：

### 收斂性
- **SARSA**：更快達到穩定的平均獎勵（約 −25 ~ −20）
- **Q-learning**：收斂至理論最優但稍低的平均獎勵（約 −45 ~ −35）

### 策略差異
- **SARSA** 學到上方繞道路徑（保守，遠離懸崖）
- **Q-learning** 學到貼近懸崖的捷徑（激進，理論最短）

### 穩定性
- **SARSA**：波動較小，因為探索時的懸崖風險已被內化於Q值
- **Q-learning**：波動較大，訓練過程中探索導致反覆掉崖

### 情境建議
| 情境 | 建議演算法 |
|------|-----------|
| 需要理論最優策略，可接受訓練不穩定 | Q-learning |
| 需要安全穩定的行為策略 | SARSA |
| 探索風險代價極高的場景 | SARSA |
| 目標是純粹的理論分析 | Q-learning |
